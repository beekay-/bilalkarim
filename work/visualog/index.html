Visualog

An experiment at visualizing a year-long journey using Google Maps, with data from Moves & Instagram.

Visit Site

Travel curiosity

While traveling around cities, where do I spend my time, and how do I get to where I’m going? What movement patterns could be seen from a year’s worth of journey?

Movement tracking

Track my location using Moves to assemble a year-long dataset of different movements and places visited for understanding where and how I travel around cities.

Process

Since location plays an important role in determining my curiosity, the UI of the web app had to be map-centric. Furthermore, the UI needed to have good contrast between the background (map) and foreground (data layers). At the end of 2017, I exported the entire collection of movement and places data from Moves in GeoJSON format and imported it into MongoDB. The total number of days I tracked myself was 341. Most of the core web app functionality was built using Google Maps API and Flask. The web app itself was served using Apache on the Google Cloud Platform.

Challenges

Several challenges—from continually tracking myself to application performance—were experienced during the development of this project. Additionally, good contrast is needed to highlight the data. Initially I wanted the map style to reflect time of the day, but struggled with technical development.

Constant tracking. Using Moves on an iPhone 7 to constantly track myself was challenging. Firstly, it greatly affected the device battery life so I had to constantly manage location tracking by turning it off and on. Secondly, I had to get into the habit turning location tracking on if it was off in the first place.

App performance. 341 days of movement and places visited was initially a heavy dataset and slow to load. I struggled with loading all of the data on the map at once. After trying multiple techniques, I found success in sending one request to fetch data instead of two, and by using GZIP compression on the server-side.

Future ideas

Some ideas could not make it to the initial release of this project due to technical and cost limitations, but will be worth exploring in future.

Street view timelapse. I initially wanted to play certain routes with a start and end point in the form of a timelapse using Google Street View, but ran into API and cost limitations.

Multi-year browsing. Furthermore, I’d love to be able to browse multi-year movement and places data to see how my travel patterns change with time.

Deeper photos integration. Currently, Instagram photos exist on a separate page, but it would be better to incorporate them into the map as well. Deeper photos integration with the map can help with telling a complete story of a particular journey.

Natural language querying. Using some of the natural language parsing technologies available as part of the Google Cloud Platform, I’d love to be able to just ask the web app where I was on a particular day and time, and have it present that information to be on the map.
